Current system

8-Agent Debate Club

THEREFORE — here's the new architecture
Part 3 — The Mechanism

Two halves.
One fast system.

Split the work into what gets paid once (the evidence library) and what gets paid per client (a fast compile pass). The expensive reasoning never touches the critical path again.

Offline Library
Science atoms verified once. Strong models do the slow work here — not per client.
Per-Client Compile
Intake → retrieve → plan-graph → one arbitration → render. Minutes, not hours.
Fast Model Tier
Gemini Flash + Sonnet handle parse, retrieve, render. Fast models never invent citations.
Strong Model Tier
Opus + GPT-5.5 reserved for arbitration and high-stakes safety only.

Evidence Library

OFFLINE · AMORTIZED
1
Verified atoms — each with citations, population-applicability ranges, and falsification signals.
2
Slow reasoning here — Opus + GPT-5.5 do the hard verification work once, offline.
3
Git-versioned files + derived DB + embeddings. Tiered human sign-off before release.
4
Cost amortized across every future client. Pay once, reuse infinitely.

Per-Client Compiler

RUNTIME · MINUTES
1
Intake → ClientState — structured object parsed by a fast model.
2
Retrieve matching atoms from the library via semantic embeddings.
3
PLAN-GRAPH compile — dependency DAG over ClientState; one node per domain.
4
1× strong-model arbitration over the compressed graph — targeted critics only where risk is high.
5
Deterministic render → two HTML consoles (diet + training).

Model Routing

FAST / STRONG SPLIT
F
Fast tier (Gemini Flash, Sonnet 4.6) — parse, retrieve, render, patch. Speed-first.
S
Strong tier (Opus 4.7, GPT-5.5) — arbitration + high-stakes safety decisions only.
!
Guardrail: fast models may select, summarize, and render — but never invent citations or override constraints.

Dynamic Re-planning

DAG · CHANGE LEDGER
1
Plan = DAG over ClientState — each node hashed to its inputs.
2
Change invalidates only dependent nodes. Knee injury → lower-body + safety + calendar. NOT protein target.
3
Change Ledger falls out of node-hash diffs for free — full audit trail at no extra cost.
4
Re-plan in seconds (projected), not hours — because only the invalidated subgraph recomputes.
